# Quality Check Process

このプロジェクトでは、公開前に品質チェックだけを行う工程を必ず挟む。

## 目的

継続的に利用者を増やすため、以下の品質を分けて確認する。

- 情報品質: 公式ソース、確認日、価格単位、注記
- 計算品質: 入力値、境界値、割引、為替換算
- 表示品質: レイアウト、読みやすさ、モバイル、横スクロール
- 信頼品質: アフィリエイト開示、注意書き、誇張表現の排除

## 自動チェック

PowerShellで最低限の構成を検査する。

```powershell
.\scripts\quality-check.ps1
```

検査内容:

- 必須ファイルが存在する
- 主要セクションと入力要素がHTMLに存在する
- `data/pricing.json` がJSONとして解析できる
- 各モデルに必須項目がある

## 目視チェック

公開前に以下を確認する。

1. 文字化けがない
2. ヒーローで目的が3秒以内に理解できる
3. 計算機がファーストビュー付近から使える
4. 比較表が横スクロール可能
5. モバイルでナビ、入力、表が破綻しない
6. 公式ソースへのリンクが見える
7. 注意書きが見える
8. アフィリエイトリンクがある場合は開示がある

## デザイン判断基準

このサイトは娯楽サイトではなく、開発者・事業者向けの意思決定補助ツールである。

優先順位:

1. 信頼性
2. 計算しやすさ
3. 比較しやすさ
4. 更新されている印象
5. 視覚的な独自性

装飾より、情報の位置、余白、表の読みやすさ、公式ソースの明示を優先する。
